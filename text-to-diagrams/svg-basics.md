# SVG 学习笔记

## 1. SVG 是什么

**SVG（Scalable Vector Graphics）= 可缩放矢量图形。**

它是一段 XML 文本，描述"画什么"，不是"像素是什么颜色"。

```html
<svg viewBox="0 0 200 100" xmlns="http://www.w3.org/2000/svg">
  <rect x="20" y="20" width="60" height="40" fill="blue"/>
</svg>
```

这段说："在一个 200×100 的虚拟坐标系里，画一个左上角在 (20,20)、宽 60、高 40 的蓝色矩形。"

跟 PNG/JPG 的根本区别：

| | 位图（PNG/JPG） | 矢量（SVG） |
|---|---|---|
| 存的是 | 每个像素的颜色值 | 几何图形的描述 |
| 放大 | 糊（像素拉伸） | 不糊（几何重新计算） |
| 文件大小 | 跟分辨率挂钩 | 跟图形复杂度挂钩 |
| 可编辑 | 改像素 | 改 XML 文本 |
| 适合 | 照片、复杂渐变 | 图标、图表、插画 |

## 2. viewBox：坐标系

### 语法

```html
<svg viewBox="min-x  min-y  width  height">
```

四个数字：

| 参数 | 含义 |
|---|---|
| `min-x` | 坐标系左边界的 x 值 |
| `min-y` | 坐标系上边界的 y 值 |
| `width` | 坐标系总宽度 |
| `height` | 坐标系总高度 |

### 例子

```html
<svg viewBox="0 0 960 460">
```

建立的坐标系：

```
  (0,0) ─────────────────────────── (960, 0)
    │                                   │
    │         可绘制的画布区域           │
    │           960 × 460               │
    │                                   │
  (0,460) ─────────────────────────── (960,460)
```

所有子元素的坐标都在这个 960×460 的虚拟空间里。

### viewBox 的关键作用

**定义"内部坐标系的单位"**——但**不决定实际显示多大**。

实际显示尺寸由 CSS 的 `width` / `height` 控制，浏览器自动等比缩放。这个分离是 SVG 最核心的设计。

## 3. 坐标系的两个特点

### 原点在左上角

不是数学课的左下角。

### y 轴向下为正

```
  (0,0) ─────→ x（向右增大）
    │
    │
    ↓
    y（向下增大）
```

所以：
- y 数字小 = 屏幕上靠**上**
- y 数字大 = 屏幕上靠**下**

举例（timeline 模板）：
- 横轴在 `y=228`
- "上方"事件标签 `y=128`（数字小，靠上）
- "下方"事件标签 `y=288`（数字大，靠下）

刚开始很容易搞反，记住一句话：**y 像水龙头的水，从上往下流。**

## 4. 常用图元

### `<rect>` — 矩形

```html
<rect x="80" y="100" width="160" height="64" 
      rx="6"
      fill="#ffffff" stroke="#141413" stroke-width="1"/>
```

| 属性 | 含义 | 默认 |
|---|---|---|
| `x` | 左上角 x | 0 |
| `y` | 左上角 y | 0 |
| `width` | 宽度 | 必填 |
| `height` | 高度 | 必填 |
| `rx` | 圆角 x 半径 | 0 |
| `ry` | 圆角 y 半径 | =rx |

几何关系：

```
   x=80                       x=240 (=80+160)
    ┌─────────────────────────┐  y=100
    │                         │
    │   width=160             │  height=64
    │                         │
    └─────────────────────────┘  y=164 (=100+64)
```

记忆口诀：
- 矩形右边 = `x + width`
- 矩形底边 = `y + height`
- 矩形中心 = `(x + width/2, y + height/2)`

### `<line>` — 直线

```html
<line x1="60" y1="228" x2="900" y2="228" 
      stroke="#b8b7b0" stroke-width="1.4"/>
```

从 `(x1, y1)` 画到 `(x2, y2)`。**没有 fill 概念**，只有 stroke。

水平线：y1 = y2
垂直线：x1 = x2

### `<circle>` — 圆

```html
<circle cx="140" cy="228" r="5" fill="#6b6a64"/>
```

- `cx, cy` = 圆心坐标
- `r` = 半径

### `<text>` — 文字

```html
<text x="140" y="143" 
      fill="#6b6a64" 
      font-size="12" 
      font-family="serif"
      text-anchor="middle">Concept</text>
```

| 属性 | 含义 |
|---|---|
| `x, y` | **文字基线**的位置（不是左上角！） |
| `font-size` | 字号 |
| `font-family` | 字体栈 |
| `text-anchor` | 水平对齐：`start` / `middle` / `end` |
| `fill` | 文字颜色 |

⚠️ **`y` 是基线**：字母的底边对齐线，**大写字母会向上突出**约 `font-size × 0.8`。所以排版时 y 要往下加一点点。

`text-anchor="middle"` 让 x 是文字的**水平中点**，特别适合居中标签。

### `<path>` — 路径

最强大的元素，能画任意形状。用 `d` 属性描述路径：

```html
<path d="M 360 268 L 200 268 L 200 332" 
      fill="none" stroke="#504e49"/>
```

`d` 里的命令：

| 命令 | 含义 |
|---|---|
| `M x y` | Move to（移动到） |
| `L x y` | Line to（画直线到） |
| `H x` | 水平线到 |
| `V y` | 垂直线到 |
| `C x1 y1 x2 y2 x y` | 三次贝塞尔曲线 |
| `Z` | 闭合路径 |

大写 = 绝对坐标，小写 = 相对坐标（相对于上一点）。

举例：`M 360 268 L 200 268 L 200 332` 意思 = 移动到 (360,268)，画线到 (200,268)，再画线到 (200,332)——画了一条**L 形折线**。

### `<polygon>` — 多边形

```html
<polygon points="430,216 500,242 500,268 430,320 360,268 360,242"
         fill="#EEF2F7" stroke="#1B365D"/>
```

`points` 是顶点列表，自动闭合。例子是个**菱形**（6 个顶点的话其实是不规则六边形——这里是 flowchart 的决策框）。

## 5. `<defs>` 和 `<marker>`：可复用的定义

```html
<svg viewBox="0 0 960 460">
  <defs>
    <marker id="arrow" markerWidth="8" markerHeight="6" 
            refX="7" refY="3" orient="auto">
      <polygon points="0 0, 8 3, 0 6" fill="#504e49"/>
    </marker>
  </defs>
  
  <line x1="0" y1="0" x2="100" y2="0" 
        stroke="#504e49" 
        marker-end="url(#arrow)"/>
</svg>
```

`<defs>` 里放**定义但不直接绘制**的东西。`<marker>` 是箭头标记。`marker-end` 让线的末端长出箭头。

类似的还有：
- `<pattern>` — 重复填充图案（timeline 的点阵背景就是用它）
- `<linearGradient>` — 线性渐变
- `<clipPath>` — 裁切区域

定义之后通过 `id` + `url(#id)` 引用，**可以多次复用**。

## 6. 颜色与描边

### `fill` 和 `stroke`

| 概念 | 含义 |
|---|---|
| `fill` | **填充**色（图形内部） |
| `stroke` | **描边**色（图形轮廓） |
| `stroke-width` | 描边粗细 |
| `fill="none"` | 不填充（透明） |
| `stroke="none"` | 没有描边 |

⚠️ **rect 默认 `fill="black"` 而不是透明！** 不写 fill 的矩形是实心黑色。要透明必须显式 `fill="none"` 或 `fill="transparent"`。

### `stroke-dasharray` — 虚线

```html
<line stroke-dasharray="4 3" .../>
```

意思：画 4 单位实线，跳 3 单位空白，循环。`"4 3"` 是常见的**短虚线**。

### `opacity` — 整体透明度

```html
<rect opacity="0.55" .../>     <!-- 整个矩形 55% 不透明 -->
<rect fill-opacity="0.3" .../> <!-- 只填充透明，描边不透明 -->
```

### 颜色写法

```html
fill="#1B365D"                    <!-- 16 进制 -->
fill="rgb(27, 54, 93)"            <!-- RGB -->
fill="rgba(27, 54, 93, 0.5)"      <!-- 带透明度 -->
fill="blue"                       <!-- 命名颜色 -->
```

## 7. 画家算法：顺序就是层级

SVG **没有 `z-index`**。元素按照在 XML 里出现的顺序，**从下往上一层层叠**——后写的覆盖先写的。

```html
<rect fill="red" .../>     ← 最先画，在最底层
<rect fill="blue" .../>    ← 中层，盖住红色
<text>HELLO</text>           ← 最后画，盖在最上面
```

这就像油画家从背景往前景画。

实战影响：

```html
<!-- ❌ 错：文字被矩形盖住，看不见 -->
<text x="100" y="50">Label</text>
<rect x="80" y="30" width="80" height="40" fill="white"/>

<!-- ✓ 对：先画矩形，再画文字 -->
<rect x="80" y="30" width="80" height="40" fill="white"/>
<text x="100" y="50">Label</text>
```

典型的画图顺序（timeline 模板就这样）：

1. 背景 rect（铺满）
2. 装饰 pattern（点阵）
3. 辅助线（横轴）
4. 连接线（虚线）
5. 节点形状（圆、矩形）
6. 标签框
7. 文字

## 8. CSS width vs viewBox：缩放的两个层次

这是 SVG 最容易搞混、也最强大的设计。

### 两层尺寸

```html
<svg viewBox="0 0 960 460" 
     style="width: 100%; min-width: 860px;">
```

| 概念 | 是什么 | 控制方式 |
|---|---|---|
| **viewBox** | 内部虚拟坐标系 | XML 属性 |
| **CSS width** | 屏幕上实际显示多宽 | CSS（也可以是 HTML attr） |

两者**完全独立**。viewBox 永远是 960×460，CSS 控制把这个画布画多大。

### 缩放公式

```
缩放因子 = 实际显示宽 / viewBox 宽
        = CSS_width / 960
```

| 实际显示 | 缩放因子 | viewBox 内 x=444 在屏幕的位置 |
|---|---|---|
| 480px | 0.5 | 222px |
| 960px | 1.0 | 444px（1:1） |
| 1920px | 2.0 | 888px |

无论缩放多少，`x="444"` 这个数字不变——变的是浏览器把每个坐标单位画成多大。

### `width: 100%; min-width: 860px;` 怎么读

```css
svg { width: 100%; min-width: 860px; }
```

- `width: 100%` = 撑满父容器
- `min-width: 860px` = 但是不能小于 860px

| 父容器 | width:100% 想要 | min-width 强制 | 实际显示 |
|---|---|---|---|
| 1200px | 1200px | ≥860 ✓ | 1200px |
| 900px | 900px | ≥860 ✓ | 900px |
| 600px | 600px | ≥860 ✗ 强制 860 | 860px（**横向溢出**） |

最后一行：父容器只有 600px，但 SVG 强行 860px——**会出现横向滚动条**。

## 9. 缩放：几何完美 ≠ 可读性完美

⚠️ 容易误解的一点：SVG 缩放**几何完美**，但**可读性会随缩放损失**。

### 几何上无损

不管缩到多大多小，元素之间的：
- 比例
- 间距比
- 重叠关系

**完全保持**。该不重叠的永远不重叠。

### 可读性会损失

**字号、描边、间距的绝对像素值跟着缩放。** 当缩到太小：

| 项 | 原值（viewBox 单位） | 缩到 0.5 后实际像素 | 后果 |
|---|---|---|---|
| 节点名 | font-size=12 | 6px | 低于可读下限（~8px） |
| 副标签 | font-size=9 | 4.5px | 糊成一团 |
| 描边 | stroke-width=1 | 0.5px | 亚像素，渲染抖动 |
| 节点间距 | 16 单位 | 8px | 视觉上分不清 |

字符没"叠到"另一个字符上，但**每个字母自己糊成一坨**——抗锯齿在如此小的字号下失效，笔画粘在一起。

### 为什么 `min-width: 860px` 是工程兜底

设计者锁定字号 7/9/12，是预设了**1:1 渲染**（SVG 显示宽度 ≥ viewBox 宽度）。再小，字号低于可读阈值。

```
viewBox 字号 7 / SVG 显示 860px × 缩放因子
= 7 × (860/960) ≈ 6.3px → 字号 7 在 860 下的实际像素
```

6.3px 已经在边缘——再小就读不出 mono 小标签了。所以选 860 作为下限：**牺牲小屏幕的"无横向滚动"，换字号在边界上的可读性**。

### 矢量"无损"的真正含义

**几何无损 ≠ 可读无损。**

矢量缩放保的是**比例、形状、相对位置**——这些是数学层面的精确。

不保的是**绝对像素呈现**——这受**屏幕物理像素 + 人眼分辨能力**这两个**非矢量因素**约束。

## 10. 实战：怎么读 SVG

下次面对一段 SVG 代码，三步走：

### 第一步：看 viewBox

```html
<svg viewBox="0 0 960 460">
```

知道坐标系多大、原点在哪。脑子里建好画布。

### 第二步：每个图元心算几何

对每个 `<rect>`：
- 左 = x
- 上 = y
- 右 = x + width
- 下 = y + height
- 中心 = (x + w/2, y + h/2)

对每条 `<line>`：
- 起点 (x1, y1)，终点 (x2, y2)

对每个 `<circle>`：
- 圆心 (cx, cy)，半径 r

### 第三步：按出现顺序叠图层

从上往下读 XML，**先出现的在底层，后出现的覆盖前面**。脑子里像油画一样一层层叠。

## 11. 常见陷阱

### 陷阱一：`y` 是基线不是顶部

`<text y="50">` 的 50 是文字**基线**位置。要让文字"看起来"在 y=50，得想到大写字母会向上突出，可能要写 y=58 之类。

### 陷阱二：rect 不写 fill 默认是黑色

```html
<rect x="0" y="0" width="100" height="100"/>
```

这是**实心黑矩形**，不是透明矩形。透明要 `fill="none"`。

### 陷阱三：viewBox 不等于显示尺寸

`viewBox="0 0 100 100"` 不是说显示 100×100 像素。**显示尺寸要 CSS 控制**。viewBox 只是内部坐标系。

### 陷阱四：stroke 在元素**两侧**各占一半

```html
<rect x="0" y="0" width="100" height="100" 
      stroke="black" stroke-width="2"/>
```

这个矩形实际占用空间是 102×102（每边各超出 1px），不是 100×100。要精确像素对齐时要考虑这点。

### 陷阱五：没有 `<style>` 时，CSS 类不起作用

SVG 元素能用 `class="xxx"`，但 CSS 规则要写在 `<style>` 里或外部 CSS 里。不是所有 SVG 渲染器都支持外部 CSS（Figma 导入就常出问题）。**最稳妥是把样式直接写成属性**：

```html
<!-- ✓ 最兼容 -->
<rect fill="blue" stroke="black"/>

<!-- ⚠️ 依赖 CSS 渲染 -->
<rect class="my-rect"/>
```

### 陷阱六：浏览器内嵌 vs 独立 SVG 文件

- **HTML 内嵌**：可以用外部 CSS、JS、CSS 变量
- **独立 .svg 文件**（被当成图片引用）：CSS 变量、JS 都失效，必须自己包含所有样式

text-to-diagrams 的"单图 SVG 输出"就属于第二种——所以模板里所有颜色都是写死的 hex，不是 CSS 变量。

## 12. 一张图记住

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  XML 文本 ──────────────────────────────────┐       │
│  <svg viewBox="0 0 W H">  ← 内部坐标系     │       │
│    <rect x y w h fill stroke />              │       │
│    <line x1 y1 x2 y2 stroke />                │      │
│    <circle cx cy r fill />                    │      │
│    <text x y font-size>...</text>             │      │
│    <path d="M x y L x y" stroke fill="none"/> │      │
│  </svg>                                       │      │
│                                              ↓       │
│  浏览器 / 渲染器                                     │
│  ├── 按 XML 顺序逐个画（画家算法）                    │
│  ├── viewBox 缩放到 CSS width × height                │
│  └── 输出像素                                         │
│                                                     │
│  缩放：几何无损 ✓                                    │
│        可读性受字号 / 描边 / 间距的绝对像素约束 ⚠️    │
│                                                     │
└─────────────────────────────────────────────────────┘
```
