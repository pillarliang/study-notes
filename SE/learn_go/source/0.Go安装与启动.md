# Go 安装与启动

## 前置检查

- 确认芯片架构：`uname -m` — 输出 `arm64`（Apple Silicon）或 `x86_64`（Intel）
- 更新命令行工具（可选）：`xcode-select --install`

---

## 安装方式

### 方式 A：官方安装包（推荐新手）

1. 从 [go.dev/dl](https://go.dev/dl/) 下载对应架构的 `.pkg` 文件
2. 双击安装，默认路径 `/usr/local/go/bin/go`，无需手动配置 `GOROOT`
3. 验证：`go version`

### 方式 B：Homebrew（便于升级和多版本管理）

```bash
brew install go       # 安装
go version            # 验证
brew upgrade go       # 升级
```

**多版本管理**可用 asdf：

```bash
brew install asdf
asdf plugin add golang https://github.com/asdf-community/asdf-golang.git
asdf install golang 1.22.6
asdf global golang 1.22.6
```

## 环境变量配置

> Go 1.13+ 默认使用 Go Modules，不再强制依赖 GOPATH。通常只需确保 Go 的 bin 目录在 PATH 中。

### PATH

编辑 `~/.zshrc`：

```bash
# 官方 pkg 安装
export PATH="/usr/local/go/bin:$PATH"
# Homebrew 安装（Apple Silicon）
export PATH="/opt/homebrew/bin:$PATH"
```

生效：`source ~/.zshrc`

### GOPATH（可选）

不设也能正常工作。如需固定工作区：

```bash
mkdir -p $HOME/go
echo 'export GOPATH="$HOME/go"' >> ~/.zshrc
echo 'export PATH="$GOPATH/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

查看当前配置：`go env GOPATH` / `go env GOROOT`

### GOPROXY（可选）

```bash
go env -w GOPROXY=https://proxy.golang.org,direct
# 国内网络可用：
# go env -w GOPROXY=https://goproxy.cn,direct
```

## GOPATH 模式 vs Go Modules

### GOPATH 模式（旧）

所有代码必须放在 `$GOPATH/src/` 下，目录结构固定：

```
$HOME/go/
├── src/    # 源代码（含自己的和依赖包）
├── pkg/    # 编译中间文件
└── bin/    # 可执行文件（go install 生成）
```

缺点：全局依赖，项目间容易冲突。

### Go Modules（Go 1.13+ 默认）

每个项目通过 `go.mod` 独立管理依赖，项目可放在**任意目录**。

```
go.mod   # 模块名 + 依赖版本约束
go.sum   # 所有依赖的校验哈希（保证可重复构建）
```

类比：类似 Node.js 的 `package.json` 或 Python 的 `requirements.txt`。

### 对比

| 对比项 | GOPATH 模式 | Go Modules |
| --- | --- | --- |
| 项目位置 | 必须在 `$GOPATH/src` 下 | 任意目录 |
| 依赖管理 | 全局引用源码路径 | `go.mod` 独立定义版本 |
| 版本锁定 | 不支持，容易冲突 | 支持 |
| 适用场景 | 老旧项目 | 新项目（官方推荐） |

> 只有在 `$GOPATH/src` 下且没有 `go.mod` 时，才会退回旧模式。

## 第一个 Go 项目

```bash
mkdir -p ~/Dev/hello-go && cd ~/Dev/hello-go
go mod init hello-go
```

创建 `main.go`：

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go on macOS!")
}
```

运行与构建：

```bash
go run .              # 直接运行
go build -o hello     # 编译为可执行文件
./hello
```

## 依赖管理工作流

### 添加依赖（推荐方式）

直接在代码中 `import`，然后运行 `go mod tidy`，Go 自动下载并更新清单。

### 指定版本

```bash
go get github.com/fatih/color@v1.16.0
```

### 常见场景速查

| 场景 | 操作 |
| --- | --- |
| 刚 clone 项目 | `go build ./...` 或 `go test ./...`（自动下载依赖） |
| 清理/同步依赖 | `go mod tidy` |
| 添加依赖 | `import` → `go mod tidy` |
| 固定版本 | `go get 包@版本` → `go mod tidy` |
| 删除依赖 | 删除 import → `go mod tidy` 或 `go get 包@none` |

## 常用命令速查

```bash
go version            # 查看版本
go env                # 查看所有环境变量
go env -w KEY=VALUE   # 永久写入环境变量
go mod init <module>  # 初始化模块
go mod tidy           # 整理依赖
go get <pkg>          # 获取依赖
go build              # 编译
go run .              # 运行当前模块
go test ./...         # 运行所有测试
```

## 升级与卸载

**官方 pkg**：安装新版 `.pkg` 即可覆盖升级。卸载：

```bash
sudo rm -rf /usr/local/go
sudo rm -f /etc/paths.d/go
sed -i '' '/\/usr\/local\/go\/bin/d' ~/.zshrc
```

**Homebrew**：`brew uninstall go`
