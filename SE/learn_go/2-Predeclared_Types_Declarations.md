# Chapter 2：预声明类型与声明

## 核心原则

Go 的首要原则：**编写能清晰表达意图的代码**。本章涉及的类型选择、变量声明方式、命名规范都围绕这一目标展开。

---

## 1. 零值（Zero Value）

Go 为每种类型提供了默认的**零值**，声明变量但未赋值时自动使用零值。这消除了 C/C++ 中未初始化变量的 bug 来源。

| 类型 | 零值 |
|------|------|
| 数值类型（int, float 等） | `0` |
| bool | `false` |
| string | `""` （空字符串） |
| complex | `0+0i` |

---

## 2. 字面量（Literals）

Go 有四种常见字面量：整数、浮点数、rune（字符）、字符串。

### 2.1 整数字面量

默认十进制，支持不同进制前缀：

```go
42        // 十进制
0b101010  // 二进制 (0b)
0o52      // 八进制 (0o)
0x2A      // 十六进制 (0x)
1_234_567 // 可用下划线分隔提高可读性
```

> 实践建议：日常使用十进制。八进制主要用于 POSIX 权限（如 `0o777`），十六进制/二进制用于网络和位操作场景。

### 2.2 浮点数字面量

```go
3.14       // 基本形式
6.03e23    // 科学计数法
0x12.34p5  // 十六进制浮点（少见）
```

### 2.3 Rune 字面量

用**单引号**包裹，表示单个字符（与双引号的字符串严格区分）：

```go
'a'        // Unicode 字符
'\n'       // 换行符
'\t'       // 制表符
'\x61'     // 8 位十六进制
'\u0061'   // 16 位 Unicode
```

### 2.4 字符串字面量

两种形式：

```go
// 解释型字符串（interpreted）- 双引号，支持转义
"Greetings and\nSalutations"

// 原始字符串（raw）- 反引号，所见即所得，支持多行
`Greetings and
"Salutations"`
```

### 2.5 字面量是无类型的（Untyped）

字面量没有固定类型，可赋值给任何兼容类型的变量：

```go
var x float64 = 10     // 整数字面量赋给 float64，合法
var y float64 = 200.3 * 5  // 表达式中混合使用，合法
```

## 3. 预声明类型

### 3.1 布尔类型（bool）

```go
var flag bool           // 零值 false
var isAwesome = true
```

> Go **不允许**将其他类型隐式转换为 bool。不存在 "truthy/falsy" 概念，必须使用比较运算符显式判断。

### 3.2 整数类型

Go 提供 12 种数值类型，分为有符号和无符号两类：

| 类型 | 范围 |
|------|------|
| `int8` | -128 ~ 127 |
| `int16` | -32768 ~ 32767 |
| `int32` | -2147483648 ~ 2147483647 |
| `int64` | -9223372036854775808 ~ 9223372036854775807 |
| `uint8` | 0 ~ 255 |
| `uint16` | 0 ~ 65535 |
| `uint32` | 0 ~ 4294967295 |
| `uint64` | 0 ~ 18446744073709551615 |

**特殊整数类型：**

- **`byte`**：`uint8` 的别名，用于表示字节数据
- **`int`**：平台相关，32 位 CPU 上为 `int32`，64 位 CPU 上为 `int64`。**整数字面量默认类型为 `int`**
- **`uint`**：与 `int` 类似但无符号
- **`rune`**：`int32` 的别名，用于表示 Unicode 码点
- **`uintptr`**：用于存储指针值

**如何选择整数类型：**

1. 处理二进制文件或网络协议 → 使用对应大小的类型（如 `int32`, `uint16`）
2. 编写泛型库函数 → 使用泛型类型参数
3. 其他情况 → **直接用 `int`**

**整数运算符：**

```go
// 算术运算
var x int = 10
x *= 2              // 复合赋值：+=, -=, *=, /=, %=

// 比较运算
x == 20             // ==, !=, >, >=, <, <=

// 位运算
x << 2              // 左移
x >> 1              // 右移
x & 0xFF            // 按位与
x | 0x01            // 按位或
x ^ 0x10            // 按位异或
x &^ 0x01           // 按位与非（AND NOT）
```

> 注意：整数除法结果仍为整数，**向零截断**（truncation toward zero）。除以 0 会引发 panic。

```go
fmt.Println(10 / 3)    // 3
fmt.Println(-10 / 3)   // -3（不是 -4）
fmt.Println(7 / 2)     // 3
fmt.Println(-7 / 2)    // -3
```

需要浮点结果时，必须先做类型转换：

```go
var a, b int = 10, 3
result := float64(a) / float64(b)   // 3.3333333333333335
```

### 3.3 浮点类型

| 类型 | 说明 |
|------|------|
| `float32` | 约 6~7 位有效数字 |
| `float64` | 约 15~16 位有效数字（**默认类型，推荐使用**） |

```go
var f float64 = 3.14
```

**浮点数注意事项：**

- 遵循 IEEE 754 标准，存在精度问题
- **不要用浮点数表示货币等需要精确值的数据**
- 非零浮点数除以 0 得到 `+Inf` 或 `-Inf`；`0.0 / 0.0` 得到 `NaN`
- 比较浮点数时不要用 `==`，应定义一个 epsilon 值判断差值是否在可接受范围内

**正确做法：epsilon 比较**

epsilon（希腊字母 ε，意为"极小量"）表示可接受的误差范围。判断两数差的绝对值是否小于此范围：

```go
import "math"

const epsilon = 1e-9   // 容差：10⁻⁹

a := 0.1 + 0.2
b := 0.3

// ❌ 不可靠
if a == b { ... }

// ✅ 可靠
if math.Abs(a - b) < epsilon { ... }
```

**epsilon 取值因场景而定：**

- 科学计算：`1e-15`
- 一般业务：`1e-9`
- 图形渲染：`1e-6`

无通用值，需根据精度需求决定。

**需要绝对精确时（如金融金额）的替代方案：**

- 用整数存储（以分为单位而非元）
- 使用第三方十进制库（如 `shopspring/decimal`）

### 3.4 复数类型（complex）

```go
var c complex128 = complex(2.5, 3.1)  // 2.5+3.1i
fmt.Println(real(c))                   // 实部：2.5
fmt.Println(imag(c))                   // 虚部：3.1
```

- `complex64`：实部和虚部各为 `float32`
- `complex128`：实部和虚部各为 `float64`（默认类型）

> 实际开发中极少使用，主要用于科学计算场景。

### 3.5 字符串与 Rune

```go
var s string              // 零值 ""
var r rune = 'J'          // 用 rune 而非 int32 表示字符，语义更清晰

// 字符串操作
s1 := "Hello"
s2 := "World"
s3 := s1 + " " + s2      // 拼接
equal := s1 == s2         // 比较：==, !=, >, >=, <, <=
```

> 字符串是**不可变的**（immutable）：可以重新赋值变量，但不能修改字符串内容。

## 4. 显式类型转换（Explicit Type Conversion）

Go **不支持**自动类型提升（automatic type promotion）。不同类型之间的运算必须**显式转换**：

```go
var x int = 10
var y float64 = 30.2

var sum1 float64 = float64(x) + y   // int → float64
var sum2 int = x + int(y)           // float64 → int（截断小数部分）
fmt.Println(sum1, sum2)             // 输出：40.2 40
```

不同大小的整数也需要显式转换：

```go
var x int = 10
var b byte = 100

var sum3 int = x + int(b)       // byte → int
var sum4 byte = byte(x) + b     // int → byte
```

> Go 通过类型转换的显式性换取了更高的代码**清晰度和安全性**——这是 Go 的设计哲学之一。

## 5. var 与 := 声明方式

### 5.1 var 声明

```go
// 完整形式：类型 + 赋值
var x int = 10

// 省略类型（由右侧推断）
var x = 10

// 省略赋值（使用零值）
var x int

// 同时声明多个变量
var x, y int = 10, 20
var x, y int                   // 都为零值
var x, y = 10, "hello"        // 不同类型

// 声明列表
var (
    x    int
    y         = 20
    z    int  = 30
    d, e      = 40, "hello"
    f, g string
)
```

### 5.2 短声明 `:=`

`:=` 是 `var` + 类型推断的简写，**仅限函数内部使用**。三种写法等价：

```go
var x int = 10    // 完整形式
var x = 10        // 省略类型（推断为 int）
x := 10           // 短声明，省略 var
```

包级别（函数外）必须用 `var`：

```go
package main

// pkgVar := 10      ❌ 编译错误：non-declaration statement outside function body
var pkgVar = 10      // ✅ 包级别只能用 var

func main() {
    x := 10           // ✅ 函数内推荐用 :=
}
```

#### `:=` 的特殊能力：复用已存在的变量

普通规则下，`:=` 不能用于已声明的变量：

```go
x := 10
x := 20   // ❌ 编译错误：x 已声明
```

但当**左侧有多个变量**时，**只要至少一个是新变量**，其他可以是已存在的——已存在的变量被**重新赋值**，新变量被**声明**：

```go
x := 10
x, y := 30, "hello"  // ✅ y 是新变量，x 被重新赋值
// 结果：x=30, y="hello"
```

如果左侧**所有变量都已存在**，则编译失败，必须改用 `=`：

```go
x := 10
y := "hello"
x, y := 30, "world"  // ❌ no new variables on left side of :=
x, y = 30, "world"   // ✅ 改用 =
```

**实际用途：错误处理中复用 `err`**

```go
file, err := os.Open("a.txt")    // 声明 file 和 err
if err != nil { ... }

data, err := io.ReadAll(file)    // ✅ data 新声明，err 复用
if err != nil { ... }
```

否则每次都要起新名字（`err1`、`err2`），代码会很啰嗦。

**陷阱：变量遮蔽（shadowing）**

在**新作用域**里使用 `:=`，可能意外创建新变量而非复用外层变量：

```go
x := 10
if true {
    x, y := 20, 30   // ⚠️ 这里的 x 是新变量（仅在 if 作用域内），不是外层 x
    fmt.Println(x)   // 20
}
fmt.Println(x)       // 10（外层 x 没变）
```

遇到这种场景，应明确用 `var` 声明新变量后再用 `=` 赋值，避免歧义。

### 5.3 选择指南

| 场景 | 推荐方式 | 示例 |
|------|---------|------|
| 初始化为零值 | `var` + 显式类型 | `var x int` |
| 类型与字面量默认类型不同 | `var` + 显式类型 | `var x byte = 20` |
| 函数内一般赋值 | `:=` | `x := 10` |
| 包级别变量 | `var`（`:=` 不可用） | `var pkgVar = "hello"` |
| 避免变量遮蔽（shadowing） | `var` + 显式声明 | 明确区分新旧变量 |

> **最佳实践**：避免在函数外部声明值可变的包级别变量，这会使数据流分析变得困难。

## 6. 使用 const

`const` 用于声明不可变值，但 Go 的 `const` 功能比较有限：

```go
const x int64 = 10

const (
    idKey   = "id"
    nameKey = "name"
)

const z = 20 * 10       // 编译期表达式

func main() {
    const y = "hello"
    // x = x + 1         // 编译错误！常量不可修改
}
```

**const 只能赋值以下内容（编译期可确定的值）：**

- 数值字面量
- `true` 和 `false`
- 字符串
- Rune
- 内置函数 `complex`、`real`、`imag`、`len`、`cap` 的返回值
- 由上述值和运算符组成的表达式

```go
x := 5
y := 10
const z = x + y   // 编译错误！运行时计算的值不能赋给 const
```

> Go 中 `const` 是给字面量命名的方式，**无法**声明运行时不可变的变量。数组、切片、map、struct 也没有 immutable 版本。

### 有类型常量 vs 无类型常量

```go
// 无类型常量 - 更灵活，可赋给兼容类型
const x = 10
var y int = x
var z float64 = x
var d byte = x       // 全部合法

// 有类型常量 - 只能赋给对应类型
const typedX int = 10
// var f float64 = typedX   // 编译错误！
```

> 如果常量可能用于多种数值类型，保持无类型；需要强制类型约束时才用有类型常量。

#### 为什么 `var x = 10` 有类型，而 `const x = 10` 无类型

关键区别在于**存在的时机**：

|              | `var x = 10`     | `const x = 10`   |
| ------------ | ---------------- | ---------------- |
| 存在时机     | 运行时           | 编译期           |
| 占用内存     | ✅ 需要分配内存  | ❌ 不占用内存    |
| 可被修改     | ✅ 可重新赋值    | ❌ 永远不变      |

**变量必须有类型**——因为运行时真实存在于内存中，CPU 操作时必须知道：

- 占多少字节（`int` 8 字节 vs `byte` 1 字节）
- 怎么解读这堆 bit（整数？浮点？字符串指针？）
- 能做什么运算（整数除法 vs 浮点除法的 CPU 指令不同）

**常量可以无类型**——因为它在编译期被**直接替换**到使用位置，运行时根本不存在：

```go
const x = 10
var a int = x        // 编译器把这里直接替换为 var a int = 10
var b float64 = x    // 编译器把这里直接替换为 var b float64 = 10
var c byte = x       // 编译器把这里直接替换为 var c byte = 10
```

每个使用点用 `10` 这个字面量去**适配当前上下文**——所以不需要固定类型。

**类比理解：**

- 变量 = 仓库里的货物。仓库要建多大、用什么材料，必须**提前决定**（类型）
- 常量 = 设计图纸上的标注。需要的地方就**复制一份**，每个使用点按需调整

> **核心规则：变量在运行时占内存，必须有类型才能存取；常量在编译期被替换为值，无需固定类型**——这就是字面量、无类型常量能"适配"多种类型，而变量不能的根本原因。

## 7. 未使用的变量与常量

### 未使用的局部变量 → 编译错误

```go
func main() {
    x := 10        // 首次赋值未被读取（编译器不报错）
    x = 20
    fmt.Println(x) // 只要变量被读取过一次就不报错
    x = 30         // 最后一次赋值未被读取（编译器不报错）
}
```

> 编译器只检查变量是否至少被**读取**过一次。第三方工具（如 `go vet`、静态分析器）可以检测未被读取的赋值。

### 未使用的常量 → 不报错

常量在编译期计算且无副作用，未使用的常量不会编入二进制文件。

## 8. 命名规范

### 基本规则

- 标识符以字母或下划线开头，可包含字母、数字、下划线
- Go 的"字母"和"数字"涵盖所有 Unicode 字符（但不推荐使用非 ASCII 字符）

### 惯用风格

| 规则 | 说明 |
|------|------|
| **驼峰命名** | `indexCounter`、`numberTries`（不用蛇形 `index_counter`） |
| **不用全大写** | 常量也用驼峰：`maxRetries` 而非 `MAX_RETRIES`，因为首字母大写在 Go 中有导出语义 |
| **短作用域用短名** | `for` 循环中用 `i`、`k`、`v` 等 |
| **大作用域用描述性名** | 包级别变量/常量使用完整名称，但仍省略类型前缀 |
| **`_` 是特殊标识符** | 单独的下划线 `_` 用于丢弃不需要的值（空标识符） |

```go
// 好的命名
func main() {
    for i := 0; i < 10; i++ { ... }       // 短循环用短名
}
var maxConnections int                      // 包级别用描述性名

// 避免的命名
var _0 = 0_0       // 滥用下划线和 Unicode
var ａ = "hello"    // 使用全角字符，容易混淆
```

> 首字母大小写决定可见性：大写开头 = 包外可访问（导出），小写开头 = 包内私有。
