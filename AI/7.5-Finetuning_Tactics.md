# Finetuning Tactics（微调实践策略）

> **核心观点**：微调过程本身并不难——框架已经封装了大部分训练逻辑，甚至能推荐默认超参数。真正需要决策的是三件事：**选择基座模型、选择微调方法、选择微调框架**。此外，几个关键超参数（学习率、批大小、训练轮数、Prompt Loss Weight）的调优直接影响微调效果。

---

## 一、基座模型选择

### 1.1 渐进式路径（Progression Path）

来自 OpenAI 微调最佳实践文档的建议流程：

| 步骤 | 目标 | 使用的模型 |
|------|------|-----------|
| 1 | **验证代码**——确保训练流程跑通 | 最便宜、最快的模型 |
| 2 | **验证数据**——观察 loss 是否随数据量下降 | 中等规模模型 |
| 3 | **探索上限**——看最好的模型能到什么效果 | 最强模型 |
| 4 | **映射性价比边界**——用所有模型做实验，选最适合的 | 全部模型 |

### 1.2 蒸馏路径（Distillation Path）

| 步骤 | 做法 |
|------|------|
| 1 | 用**小数据集 + 最强模型**训练出最佳模型（强基座模型需要更少数据） |
| 2 | 用这个微调模型**生成更多训练数据** |
| 3 | 用新数据集训练**更便宜的模型** |

> **前置条件**：微调通常在 prompt 工程之后进行。到微调阶段，你应该已经对不同模型的行为特征有较好的理解，据此规划微调路径。

---

## 二、微调方法选择

| 考虑因素 | Adapter 方法（如 LoRA） | 全参微调 |
|----------|------------------------|---------|
| **数据量** | 几百条即可见效 | 通常需要数千条以上 |
| **成本** | 中等 GPU 即可 | 需要更多算力 |
| **性能** | 接近但通常不及全参微调 | 上限更高 |
| **多模型部署** | 共享基座 + 多个 adapter，效率高 | 每个模型独立部署 |

> **建议**：从 LoRA 开始，只有在 LoRA 效果不够时才尝试全参微调。

---

## 三、微调框架选择

### 3.1 API 方式

上传数据 → 选择基座模型 → 获取微调模型。由模型提供商、云服务商或第三方提供。

| 优点 | 缺点 |
|------|------|
| 简单快速 | 可选基座模型有限 |
| 无需管理基础设施 | 可调超参数有限，定制空间小 |

### 3.2 开源框架（2025 更新）

| 框架 | 特点 | 适合谁 |
|------|------|--------|
| **Unsloth** | 比 Flash Attention 2 快 2-5 倍、省 80% 内存；仅支持**单 GPU** | 个人开发者、消费级硬件 |
| **LLaMA-Factory** | 算法覆盖最广（LoRA、QLoRA、DoRA、GaLore 等）；Web UI 配置 | 快速实验、广泛模型/方法覆盖 |
| **Axolotl** | 多 GPU 支持；YAML 配置驱动；支持多模态微调（beta） | 团队协作、多 GPU 训练 |
| **Torchtune**（Meta） | PyTorch 原生，零抽象开销；面向消费级 GPU（24GB） | 深度定制、PyTorch 生态 |
| **PEFT**（Hugging Face） | PEFT 方法的标准实现，集成 DoRA、PiSSA 等最新变体 | 研究和原型开发 |
| **LitGPT** | 轻量级，专注于 GPT 类模型 | 快速上手 |

### 3.3 分布式训练

如果需要多机训练，需额外使用分布式训练框架：

| 框架 | 说明 |
|------|------|
| **DeepSpeed** | 微软出品，ZeRO 优化器族 |
| **PyTorch Distributed** | PyTorch 原生分布式 |
| **ColossalAI** | 国产分布式训练框架 |

## 四、关键超参数

### 4.1 学习率（Learning Rate）

学习率决定每步训练中参数的**更新幅度**。

- **类比**：如果学习是走向目标的过程，学习率就是步幅
  - 步幅太小 → 到达目标太慢
  - 步幅太大 → 可能跨过目标，模型无法收敛

**实用指引**：

| 方面 | 建议 |
|------|------|
| 搜索范围 | 通常在 **1e-7 到 1e-3** 之间 |
| 常见做法 | 取预训练阶段末尾的学习率 × 0.1~1.0 的常数 |
| 诊断方法 | Loss 曲线波动大 → 学习率太大；Loss 曲线稳定但下降慢 → 太小 |
| 进阶策略 | **学习率调度（Learning Rate Schedule）**：训练初期用大学习率，后期逐渐缩小 |

### 4.2 批大小（Batch Size）

批大小决定模型每步从多少条样本中学习。

| 批大小 | 效果 |
|--------|------|
| 太小（< 8） | 训练不稳定——每步更新的信号噪声比太高 |
| 越大 | 更新越稳定、训练越快 |
| 越大 | **但内存需求越高**——受硬件限制 |

> **梯度累积（Gradient Accumulation）**：当内存限制只允许小批大小时，可以跨多个小批次**累积梯度**，等积累到足够多后再更新模型权重。效果等价于使用更大的批大小，但不增加内存。

### 4.3 训练轮数（Epochs）

一个 epoch = 遍历一次全部训练数据。

| 数据集规模 | 建议轮数 |
|-----------|---------|
| 百万级样本 | 1-2 轮可能足够 |
| 千级样本 | 4-10 轮仍可能有提升 |

**诊断方法**：

| 观察 | 判断 |
|------|------|
| 训练 loss 和验证 loss 都在稳定下降 | 可继续训练（增加轮数或数据） |
| 训练 loss 下降但验证 loss 上升 | **过拟合**——应减少轮数 |

### 4.4 Prompt Loss Weight

用于指令微调。每条样本由 **prompt + response** 组成，两者都可以贡献到训练 loss。

| 权重值 | 含义 |
|--------|------|
| 100% | prompt 对 loss 的贡献与 response 相同——模型同等程度地从两者学习 |
| 0% | 模型**只从 response 学习** |
| **10%（默认）** | 模型**主要从 response 学习**，但也少量从 prompt 中学习 |

> **逻辑**：推理时 prompt 由用户提供，模型只需生成 response。因此 response token 应该对训练 loss 贡献更多。

## 五、核心 Takeaway

```
┌──────────────────────────────────────────────────────────┐
│                  微调实践决策树                             │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1. 选基座模型                                            │
│     ├── 先用最强模型验证可行性                              │
│     ├── 再用全系列模型找性价比最优                          │
│     └── 或走蒸馏路径：强模型 → 生成数据 → 训练便宜模型      │
│                                                          │
│  2. 选微调方法                                            │
│     ├── 默认用 LoRA（数据少、算力有限）                     │
│     └── 效果不够再试全参微调（数据多、算力充足）            │
│                                                          │
│  3. 选框架                                                │
│     ├── 快速实验 → LLaMA-Factory / Unsloth                │
│     ├── 多 GPU → Axolotl                                  │
│     ├── 深度定制 → Torchtune                               │
│     └── 不想管基础设施 → 微调 API                          │
│                                                          │
│  4. 调超参数                                              │
│     ├── 学习率：1e-7 ~ 1e-3，看 loss 曲线调               │
│     ├── 批大小：越大越稳定，受内存限制                      │
│     ├── 轮数：小数据集多轮，看验证 loss 防过拟合            │
│     └── Prompt Loss Weight：默认 10%                      │
└──────────────────────────────────────────────────────────┘
```
