# 学习笔记：Agent 工具设计的第三代演进 (Advanced Tool Use)

**背景与演进**：
Agent 调用外部工具的设计经历了三个阶段：
* **第一代（API 封装）**：直接扔给模型底层 API，粒度太细，导致模型规划困难。
* **第二代（ACI 接口）**：Agent-Computer Interface，将工具对齐 Agent 的目标，合并繁琐操作（如直接提供 `create_script` 而不是分开提供建文件、写内容、改权限的工具）。
* **第三代（Advanced Tool Use）**：在 ACI 的基础上，**进一步优化工具的发现、调用和描述方式**，大幅提升准确率并降低 Token 成本。

第三代演进主要包含以下三个核心方向：

---

## 1. Tool Search（动态工具发现）

**核心理念**：按需取用，避免上下文（Context）过载。

* **痛点**：过去习惯将所有工具的定义（JSON Schema）一次性塞进大模型的系统提示词中。当工具数量增多时，不仅消耗大量 Token，还会导致“注意力稀释”，使得模型眼花选错工具。
* **解决方案**：系统只提供一个类似 `search_tools(query)` 的总控接口。Agent 根据当前任务，先用自然语言搜索需要的工具，后台仅将匹配到的工具详细定义注入到上下文中。
* **底层实现机制（保障搜索准确率）**：
    1.  **向量检索 (Vector Search)**：将工具描述转化为向量，通过语义相似度匹配（解决同义词/说法不一致的问题）。
    2.  **混合检索 (Hybrid Search)**：结合关键词匹配（如 BM25），确保专有名词或特定 ID 被精准命中。
    3.  **描述工程 (Description Engineering)**：这是最关键的一环。工具描述必须包含清晰的**触发条件 (Use when)**和**排他条件 (Don't use when)**。
    4.  **自主试错 (Agentic Retry)**：若首次搜索未找到合适工具，大模型具备自主修改 Query 重新搜索的能力。
* **收益与数据**：
    * 上下文保留率可达到 95%。
    * Claude 3 Opus 模型调用准确率从 **49% 飙升至 74%**。

---

## 2. Programmatic Tool Calling（代码编排）

**核心理念**：中间数据不上报，交给执行环境流转。

* **痛点**：传统模式下，Agent 像“微操大师”，每调用一步工具，工具就把长篇大论的中间结果（如几万字的网页源码、长篇 PDF 内容）传回给大模型。大模型提取后再调用下一个工具，导致海量中间数据在网络和 LLM 之间来回穿梭。
* **解决方案**：大模型直接输出一段包含多个工具调用的**编排代码（如 Python 脚本）**。这些代码在安全的沙盒环境中一次性执行，中间产生的数据完全在本地变量中流转，**不进入 LLM 的上下文**，最后只把精简的最终结果（如一个总结、一个下载链接）返回给模型。
* **收益与数据**：
    * 极大加快执行速度，减少 API 交互轮次。
    * Token 消耗呈现指数级下降，以长任务为例，消耗可从 **约 150,000 Token 骤降到 约 2,000 Token**。

---

## 3. Tool Use Examples（示例驱动）

**核心理念**：不仅要给说明书，还要“打个样”（Few-shot Learning）。

* **痛点**：定义工具参数时，通常只用 JSON Schema（如 `type: string`）。但这只能描述数据结构，无法传递具体的**业务格式要求**。例如，颜色参数需要 `#FF0000` 还是 `red`？这往往导致参数校验报错。
* **解决方案**：在工具的 JSON 定义中，显式加入 `examples` 字段，附带 **1-5 个真实的、正确的调用示例**。
* **示例代码片段**：
    ```json
    {
      "name": "create_ppt",
      "description": "生成图表幻灯片",
      "parameters": { ... },
      "examples": [
        {
          "input": {
            "chart_type": "pie",
            "theme_color": "#0055FF"  // 通过示例明确告知需要 HEX 格式
          }
        }
      ]
    }
    ```
* **收益与数据**：
    * 充分利用了大模型强大的少样本（Few-shot）模仿能力，显著减少了格式幻觉。
    * 工具调用准确率从 **72% 提升到 90%**。

---

## 总结

第三代 Agent 工具使用的本质是**“工程化减负”**：
1.  **Tool Search** 减少了模型**阅读**的负担（剔除无关干扰）。
2.  **Programmatic Tool Calling** 减少了模型**记忆**的负担（剔除中间冗余数据）。
3.  **Tool Use Examples** 减少了模型**猜测**的负担（消除参数格式歧义）。

掌握这三点，是开发工业级、高稳定性、低成本 AI Agent 系统的必经之路。
