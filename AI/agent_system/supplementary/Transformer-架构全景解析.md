# Transformer 架构全景解析

> **来源**：3Blue1Brown — *Deep Learning Chapter 5: Transformers, the tech behind LLMs*
>
> **核心定位**：本文从第一性原理出发，以可视化方式拆解 Transformer 内部的数据流动过程。重点覆盖：GPT 的宏观结构、Token 化、词嵌入（Embedding）、反嵌入（Unembedding）、Softmax 与温度采样。注意力机制的详细计算见 [self-attention-自注意力机制](self-attention-自注意力机制.md)。

## 一、GPT 是什么——三个词拆解

**GPT = Generative Pre-trained Transformer**

| 缩写 | 含义 | 说明 |
|---|---|---|
| **Generative** | 生成式 | 模型能够生成新文本，而非仅做分类或标注 |
| **Pre-trained** | 预训练 | 模型先从海量数据中学习通用知识；"预"字暗示后续可通过**微调**（fine-tuning）适配特定任务 |
| **Transformer** | 一种神经网络架构 | 2017 年由 Google 在论文 *"Attention Is All You Need"*（Vaswani et al.）中提出，是当前 AI 热潮的核心发明 |

> **关键认识**：Transformer 不仅用于文本。基于它可以构建：语音识别（音频→文字）、语音合成（文字→音频）、文生图（DALL·E、Midjourney）、机器翻译等多种模型。ChatGPT 背后的变体是专门用来**预测下一个 token** 的自回归语言模型。

## 二、自回归生成——"预测下一个词"如何变成"写文章"

### 2.1 核心循环

Transformer 语言模型本质上是一个**条件概率预测器**：给定一段文本，输出"下一个 token"的概率分布。

```
输入文本 → Transformer → 概率分布 P(next_token | context)
```

要生成更长的文本，只需反复执行以下循环：

1. 将当前文本输入模型，得到下一个 token 的概率分布
2. 从该分布中**采样**一个 token
3. 将采样结果拼接到文本末尾
4. 重复步骤 1

> **直觉检验**：这种方法看似简单到不应该奏效。视频中演示了 GPT-2（较小模型）生成的故事毫无逻辑，但换成 GPT-3（同架构、参数量大得多）后，突然能生成连贯且有推理能力的故事。这正是**规模定律**（Scaling Law）的体现——模型足够大时，"预测下一个词"这一简单目标涌现出惊人的能力。

### 2.2 从补全器到聊天机器人

将"下一词预测器"变成聊天机器人的最简单方式：

1. 准备一段**系统提示词**（System Prompt），设定"用户正在与一个有帮助的 AI 助手对话"的场景
2. 将用户的问题作为对话开头
3. 让模型预测"这样一个有帮助的 AI 助手会如何回应"

> GPT-3 最早的公开演示就是纯文本补全，而非对话形式。从补全到对话，还需要额外的训练步骤（如 RLHF），详见 [Post-Training-后训练详解](Post-Training-后训练详解.md)。

## 三、数据在 Transformer 中的宏观流动

### 3.1 全流程概览

```
原始文本
  ↓ ① Token 化
Token 序列 [t₁, t₂, ..., tₙ]
  ↓ ② 嵌入（Embedding）
向量序列 [e₁, e₂, ..., eₙ]     ← 每个向量编码单个 token 的含义
  ↓ ③ 注意力模块（Attention Block）  ×N 层
向量序列（已融合上下文）           ← 向量之间相互交流，更新含义
  ↓ ④ 前馈层 / MLP（Feed-Forward）  ×N 层
向量序列（进一步精炼）             ← 向量独立地经过相同运算
  ↓ ⑤ 反嵌入（Unembedding）
Logits 向量（词汇表大小）
  ↓ ⑥ Softmax
概率分布 P(next_token)
```

> **注意力模块**和**前馈层**交替堆叠、反复循环，形成 Transformer 的"层"（Layer）。GPT-3 有 **96 层**。

### 3.2 两类核心模块的职责

| 模块 | 向量间是否交互 | 职责 |
|---|---|---|
| **注意力模块**（Attention） | **是**——向量之间相互传递信息 | 弄清上下文中哪些词与哪些词相关，据此更新向量的含义。例如 "a machine learning **model**" 与 "a fashion **model**" 中 "model" 应有不同表示 |
| **前馈层 / MLP** | **否**——向量独立、并行处理 | 对每个向量提出一长串"问题"，根据答案更新向量。可理解为对每个位置的知识做进一步提炼 |

> **所有实际计算几乎都是矩阵乘法**。模型中的矩阵就是**权重**（weights），即训练过程中学到的参数，决定了模型的行为。

### 3.3 权重 vs 数据——必须画清的分界线

| 类别 | 性质 | 来源 |
|---|---|---|
| **权重**（Weights） | 模型的"大脑"，决定行为 | 训练过程中从数据学习得到 |
| **被处理的数据**（Activations） | 某次推理的特定输入内容 | 用户输入的文本/图像/音频 |

> GPT-3 的 **1750 亿**个权重被组织成约 **27,938 个矩阵**，分为 **8 个类别**。理解 Transformer 就是理解每个类别的矩阵在做什么。

## 四、Token 化（Tokenization）

### 4.1 什么是 Token

Token 是模型处理的最小单位。对于文本，token 通常是：
- 完整的单词（如 "hello"）
- 单词的一部分（如 "un" + "believe" + "able"）
- 标点符号或常见字符组合

对于图像或音频，token 则是图像的小块（patch）或声音的小片段。

### 4.2 为什么不直接用字符或完整单词

| 粒度 | 问题 |
|---|---|
| 字符级 | 序列过长，注意力计算开销 O(n²) 爆炸 |
| 完整单词 | 词汇表巨大，且无法处理未登录词（OOV） |
| **子词（Subword）** | 平衡序列长度与词汇表大小，是目前主流方案 |

> GPT-3 的词汇表大小为 **50,257** 个 token。常用的子词切分算法包括 BPE（Byte Pair Encoding）和 SentencePiece。

## 五、词嵌入（Word Embedding）

### 5.1 嵌入矩阵 W_E

模型有一个预定义的**嵌入矩阵** W_E，其每一列对应词汇表中的一个 token：

$$W_E \in \mathbb{R}^{d_{model} \times |V|}$$

- $d_{model}$：嵌入维度（GPT-3 中为 **12,288**）
- $|V|$：词汇表大小（GPT-3 中为 **50,257**）

对于输入中的每个 token，从 W_E 中**查找**对应的列向量，作为该 token 的初始表示。

> **参数量计算**：$12{,}288 \times 50{,}257 \approx$ **6.17 亿**个参数。

### 5.2 嵌入空间的语义结构

"嵌入"一个词，就是把它映射为高维空间中的一个点。训练过程中，模型会自动学习到一种嵌入方式，使得：

- **含义相近的词**在空间中彼此接近（如 "tower" 的最近邻都是与"塔"相关的词）
- **空间中的方向承载语义含义**

#### 经典类比关系示例

| 向量运算 | 结果 | 编码的语义方向 |
|---|---|---|
| king − man + woman | ≈ queen | 性别方向 |
| Italy − Germany + Hitler | ≈ Mussolini | "国家特质" + "二战领导人" |
| Germany − Japan + sushi | ≈ bratwurst | 国家文化特色 |

> 这些类比并非完美——例如 "queen" 在训练数据中的用法不仅仅是 "king 的女性版本"，所以实际嵌入会有偏差。家庭关系词通常能更好地展示这种规律。

### 5.3 点积衡量对齐程度

两个向量的**点积**（dot product）可以衡量它们的对齐程度，这一直觉对理解注意力机制至关重要：

| 情况 | 点积值 |
|---|---|
| 两个向量指向**相似方向** | 正值（越大越相似） |
| 两个向量**垂直** | 零 |
| 两个向量方向**相反** | 负值 |

> **实验验证**：取 "cats" − "cat" 作为"复数方向"向量，计算它与各词嵌入的点积。复数名词始终比对应单数名词给出更高的值。甚至 "one"、"two"、"three" 的点积值是递增的——模型可以定量衡量一个词的"复数程度"。

### 5.4 嵌入向量的上下文演化

初始嵌入仅编码**单个 token** 的含义，不包含任何上下文信息。但流经 Transformer 的各层后，每个向量会逐步**吸收上下文**：

```
初始：king（国王的通用含义）
  ↓ 经过多层注意力 + MLP
最终：编码了"一位生活在苏格兰、通过谋杀前任登位、用莎士比亚式语言描述的国王"
```

> 赋予向量"高效融合上下文的能力"是整个 Transformer 网络的**首要目标**。

## 六、上下文窗口（Context Size）

模型每次只能处理固定数量的向量，称为**上下文大小**（Context Size）。

| 模型 | 上下文大小 |
|---|---|
| GPT-3 | 2,048 tokens |
| GPT-4 | 8K / 32K / 128K tokens |
| Claude 3.5 | 200K tokens |

流经网络的数据形状始终为：**上下文大小 × 嵌入维度**（GPT-3: 2048 × 12288）。

> 上下文大小限制了模型预测下一个词时能"看到"的文本长度。这就是为什么早期 ChatGPT 在长对话中会"丢失话题"——对话长度超出了上下文窗口。

## 七、反嵌入与输出（Unembedding）

### 7.1 反嵌入矩阵 W_U

经过所有 Transformer 层后，**最后一个位置**的向量被用来预测下一个 token。反嵌入矩阵 W_U 将其映射到词汇表大小的向量：

$$W_U \in \mathbb{R}^{|V| \times d_{model}}$$

$$\text{logits} = W_U \cdot h_{\text{last}}$$

> W_U 的形状与 W_E 转置后相同（行列互换），参数量同样约 **6.17 亿**。

### 7.2 为什么只用最后一个向量

这与训练效率相关。实际上在训练时，**每个位置**的向量都会被用来预测紧随其后的 token（而不仅是最后一个），这使得每条训练样本能提供更多梯度信号，大幅提升训练效率。但在推理时，我们只关心最后一个位置的输出。

### 7.3 Logits

反嵌入矩阵输出的原始、未归一化的值称为 **logits**。词汇表中每个 token 对应一个 logit 值。Logits 可以是负数、远大于 1，且不满足概率分布的约束——需要经过 Softmax 归一化。

## 八、Softmax 函数

### 8.1 为什么需要 Softmax

深度学习的计算框架要求一切看起来像矩阵-向量乘法，但矩阵乘法的输出不满足概率分布的两个约束：
- 每个值在 [0, 1] 之间
- 所有值之和为 1

**Softmax** 是将任意实数向量转化为有效概率分布的标准方法。

### 8.2 计算公式

给定 logits 向量 $z = [z_1, z_2, ..., z_n]$：

$$\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{n} e^{z_j}}$$

**三步理解**：

1. **取指数** $e^{z_i}$：将任意实数变为正数
2. **求和** $\sum e^{z_j}$：计算所有正数的总和
3. **归一化**：每个值除以总和，确保概率之和为 1

### 8.3 Softmax 的特性

| 特性 | 说明 |
|---|---|
| **放大差异** | 最大值对应的概率接近 1，较小值接近 0 |
| **比 argmax "柔和"** | 当多个值接近时，它们都会获得有意义的概率权重 |
| **连续可微** | 输出随输入连续变化，适合梯度下降训练 |

> Softmax 不仅用于最终输出，在注意力模块中也被用来将注意力分数归一化为权重分布。

## 九、温度采样（Temperature Sampling）

### 9.1 温度参数 T

在 Softmax 的指数中引入温度参数 T：

$$\text{softmax}(z_i; T) = \frac{e^{z_i / T}}{\sum_{j=1}^{n} e^{z_j / T}}$$

| 温度 T | 效果 | 生成风格 |
|---|---|---|
| **T → 0** | 所有概率集中在最大值上（退化为 argmax） | 确定性输出，最"安全"但可能老套 |
| **T = 1** | 标准 Softmax，原始概率分布 | 平衡创意与连贯性 |
| **T > 1** | 分布更均匀，小概率词获得更多机会 | 更有创意，但可能出现不连贯内容 |

### 9.2 实际效果演示

视频中以 "Once upon a time" 为种子文本，用不同温度让 GPT-3 生成故事：

- **T = 0**：总是选最可预测的词，得到老套的"金发姑娘和三只熊"变体
- **适中 T**：故事更有创意——讲一个来自韩国的年轻网络艺术家
- **T 过高**：开头有创意但很快退化为无意义内容

> **实践注意**：OpenAI 的 API 将温度限制在 [0, 2] 范围内，防止生成过于荒谬的内容。这是工程限制而非数学限制。

## 十、GPT-3 参数量拆解

以 GPT-3（1750 亿参数）为例，追踪参数的来源：

| 组件 | 参数量 | 累计 |
|---|---|---|
| 嵌入矩阵 W_E（12288 × 50257） | ≈ 6.17 亿 | 6.17 亿 |
| 反嵌入矩阵 W_U（50257 × 12288） | ≈ 6.17 亿 | ≈ 12.34 亿 |
| 注意力模块 + MLP × 96 层 | ≈ 1737.66 亿 | ≈ 1750 亿 |

> **绝大多数参数**（约 99.3%）位于 96 层重复的注意力模块和 MLP 中。嵌入/反嵌入矩阵虽然参数量"不小"（超 10 亿），但在总量中占比不到 1%。

## 十一、深度学习的基本前提

### 11.1 机器学习的核心理念

与传统编程（显式定义规则）不同，机器学习的思路是：

1. 搭建一个具有**可调参数**的灵活结构
2. 用大量"输入→输出"示例来**自动调整**参数
3. 使模型学会模仿期望的行为

> **最简单的例子**：线性回归——2 个参数（斜率 + 截距）拟合数据点。GPT-3 是同一思想的极致——1750 亿个参数。

### 11.2 深度学习的特定格式

为了让训练算法（**反向传播**，Backpropagation）在大规模下良好运作，模型必须遵循特定格式：

| 约束 | 说明 |
|---|---|
| **输入格式化为实数数组** | 可以是一维向量、二维矩阵或更高维**张量**（tensor） |
| **逐层变换** | 数据经过许多层，每层都是实数数组→实数数组的变换 |
| **参数交互方式** | 参数与数据的交互只有**加权求和**（即矩阵-向量乘积） |
| **穿插非线性函数** | 如 ReLU、GELU 等，不依赖于参数，但赋予网络非线性拟合能力 |

> **核心洞察**：当你窥探 ChatGPT 内部时，几乎所有实际计算都是矩阵-向量乘法。理解 Transformer 就是理解这些矩阵在做什么。

## 十二、上下文窗口扩展技术（2025 更新）

原始视频基于 GPT-3 的 2048 token 上下文。自 2023 年以来，上下文窗口技术有了显著进展：

| 技术 | 说明 |
|---|---|
| **ALiBi**（Press et al., 2022） | 用线性偏置替代位置编码，天然支持外推到更长序列 |
| **RoPE 插值**（Chen et al., 2023） | 通过插值旋转位置编码（RoPE），将预训练的短上下文模型扩展到更长上下文 |
| **Ring Attention**（Liu et al., 2023） | 分布式注意力计算，理论上支持无限长上下文 |
| **实际产品** | Claude（200K）、Gemini 1.5（1M+）等模型已支持超长上下文 |

> 上下文窗口的扩展大幅缓解了"长对话丢失话题"的问题，但也带来了新的挑战：如何在超长上下文中有效检索关键信息（即"大海捞针"问题）。
