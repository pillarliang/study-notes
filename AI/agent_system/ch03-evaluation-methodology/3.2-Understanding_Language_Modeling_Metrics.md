# Understanding Language Modeling Metrics

> **核心前提**：基础模型大多以语言模型为核心组件，语言模型的性能与下游任务表现高度相关（Liu et al., 2023）。因此，理解语言模型的评估指标，是理解下游表现的基础。

---

## 四个指标的关系总览

语言模型最核心的四个指标——**交叉熵（Cross Entropy）、困惑度（Perplexity）、BPC（Bits-per-Character）、BPB（Bits-per-Byte）**——本质上是同一件事的不同表达。知道其中任何一个，都可以推算出其余三个。

```
Entropy → Cross Entropy → Perplexity
                        → BPC / BPB
```

它们自 Claude Shannon 1951 年论文以来几乎没有变过，至今仍是自回归语言模型训练的核心指导指标。

---

## 1. 熵（Entropy）

### 第一性原理

熵衡量的是：**平均来看，一个 token 携带了多少信息量**。

**信息量的本质就是"意外程度"**——越让你意外的事情，信息量越大：
- 朋友说"太阳从东边升起了" → 毫不意外 → 信息量为零
- 朋友说"太阳今天从西边升起了" → 大吃一惊 → 信息量极大

放到语言模型的语境中，熵衡量的就是：**平均每个 token 有多让模型意外。**

- 熵越高 → 每个 token 越难预测、承载的信息越多 → 需要越多 bit 来表示
- 熵越低 → 语言越有规律、越可预测

### 直觉理解

**语言场景**：
- 熵低："今天天气真__" → 大概率填"好"或"热"，选项少，容易猜
- 熵高："他昨天突然决定去__" → 可能"旅游""辞职""火星"……选项太多，很难猜

**位置描述的例子**：想象用一种"语言"描述一个正方形内的位置：

| 语言 | token 数 | 能描述的精度 | 所需 bit 数 | 熵 |
|---|---|---|---|---|
| 语言 A | 2 个 token（上/下） | 粗略 | 1 bit | 1 |
| 语言 B | 4 个 token（左上/右上/左下/右下） | 精确 | 2 bit | 2 |

语言 B 的每个 token 携带更多信息（精度更高），但代价是需要更多 bit 来编码。

**为什么"信息量高 = 需要更多 bit"？** bit 是信息的最小单位（0 或 1）。2 种可能只需 1 bit（0/1），4 种可能需要 2 bit（00/01/10/11），8 种需要 3 bit……可能性越多，每个 token 越意外，需要越多 bit 来区分。

### 与"可预测性"的关系

**熵越低，语言越容易被预测。** 如果你能完美预测我接下来要说什么，那我说的话就不携带任何新信息——信息量为零，熵为零。

## 2. 交叉熵（Cross Entropy）

### 第一性原理

训练语言模型的目标是让模型学到训练数据的分布。交叉熵衡量的是：**模型预测训练数据中"下一个 token"的难度有多大。**

### 数学分解

设 P 为训练数据的真实分布，Q 为模型学到的分布：

$$H(P, Q) = H(P) + D_{KL}(P  Q)$$

- **H(P)**：训练数据本身的熵（数据有多难预测——这是数据固有的，与模型无关）
- **D_KL(P‖Q)**：KL 散度，衡量模型分布 Q 偏离真实分布 P 的程度

### 关键推论

- 如果模型完美学到了训练数据的分布：$D_{KL} = 0$，此时 $H(P,Q) = H(P)$，交叉熵等于数据本身的熵。
- 交叉熵可以理解为模型对训练数据熵的**近似**。
- **交叉熵不对称**：$H(P,Q) \neq H(Q,P)$。
- 语言模型的训练目标就是**最小化交叉熵**。由于 $H(P)$ 是数据固有属性无法改变，最小化交叉熵本质上就是最小化 $D_{KL}$——训练语言模型的过程就是反复调整参数，让模型越来越"懂"训练数据中的规律。

## 3. BPC 与 BPB

### 问题：交叉熵没法跨模型比较

不同模型使用不同的分词方式（word-level vs. character-level vs. subword），导致"每个 token 多少 bit"这个数值没有可比性。

### 解决：统一到字符或字节级别


| 指标                          | 含义               | 计算方式                             |
| --------------------------- | ---------------- | -------------------------------- |
| **BPC**（Bits-per-Character） | 模型表示一个字符需要多少 bit | 每 token 的 bit 数 ÷ 每 token 的平均字符数 |
| **BPB**（Bits-per-Byte）      | 模型表示一个字节需要多少 bit | BPC ÷ 每字符的平均字节数                  |


### 示例

- 交叉熵 = 6 bits/token，每 token 平均 2 字符 → **BPC = 3**
- 每字符 7 bit（ASCII），即 ⅞ 字节 → **BPB = 3 ÷ (⅞) = 3.43**

### 与压缩的关系

BPB 直接反映了语言模型的**文本压缩能力**。BPB = 3.43 意味着模型能把原始文本（每字节 8 bit）压缩到原来的 3.43/8 ≈ 43%——不到一半。

> BPB 比 BPC 更标准化，因为不同字符编码（ASCII vs UTF-8）下字符的字节数不同，BPB 消除了这一差异。

## 4. 困惑度（Perplexity, PPL）

### 第一性原理

困惑度是交叉熵的指数形式：

$$PPL(P, Q) = 2^{H(P, Q)}$$

> 注意：PyTorch / TensorFlow 中使用自然对数（nat），此时 $PPL = e^{H(P,Q)}$。由于 bit 和 nat 容易混淆，很多人直接报告 PPL 而非交叉熵。

### 直觉理解

**困惑度 = 模型在预测下一个 token 时的"有效选项数"。**

- 一个完美学会 4 个位置 token 的模型，交叉熵 = 2 bit，困惑度 = $2^2 = 4$，意味着每次预测时模型需要在 4 个等可能的选项中做选择。
- PPL = 3 意味着模型每次预测相当于在 3 个选项中猜，对于词表动辄数万的模型来说，这已经是极强的预测能力。

---

## 5. 影响困惑度的三个因素


| 因素            | 影响方向   | 原因                                                                     |
| ------------- | ------ | ---------------------------------------------------------------------- |
| **数据结构化程度**越高 | PPL 越低 | 结构化数据更可预测。如 HTML 中 `<head>` 后大概率跟 `</head>`，比日常文本容易预测得多                |
| **词表越大**      | PPL 越高 | 可选 token 更多，预测更难。同一数据集上，字符级 PPL < 词级 PPL                               |
| **上下文长度越长**   | PPL 越低 | 模型拥有的线索越多，不确定性越小。Shannon 1951 年用 10 token 上下文；如今模型可用 500–10,000+ token |


---

## 6. 困惑度的实际应用

### 6.1 作为模型能力的代理指标

PPL 越低，模型的下游任务表现通常越好。GPT-2 的实验数据显示：模型越大 → PPL 越低 → 下游表现越好。


| 模型大小  | WikiText2 PPL | LAMBADA PPL |
| ----- | ------------- | ----------- |
| 117M  | 29.41         | 35.13       |
| 345M  | 22.76         | 15.60       |
| 762M  | 19.93         | 10.87       |
| 1542M | 18.34         | 8.63        |


### 6.2 检测数据污染（Data Contamination）

模型对见过并记住的文本 PPL 最低。如果模型在某个 benchmark 数据上 PPL 异常低，说明该 benchmark 可能已被纳入训练数据，评测结果不可信。

### 6.3 训练数据去重（Deduplication）

只将 PPL 较高的新数据加入训练集——如果模型已经能很好地预测某段文本（PPL 低），说明类似数据已存在，无需重复添加。

### 6.4 异常文本检测

PPL 最高的文本往往是异常文本：表达非常规想法（如"我的狗在业余时间教量子物理"）或乱码（如"home cat go eye"）。可利用这一点检测异常输出。

## 7. 困惑度的计算

给定语言模型 X 和 token 序列 $x_1, x_2, \ldots, x_n$，困惑度为：

$$PPL = \left( \prod_{i=1}^{n} \frac{1}{P(x_i | x_1, \ldots, x_{i-1})} \right)^{\frac{1}{n}}$$

其中 $P(x_i | x_1, \ldots, x_{i-1})$ 是模型给定前文后对第 $i$ 个 token 的预测概率。

> **实际限制**：计算 PPL 需要访问模型对每个 token 的概率（logprobs），但并非所有商业模型都开放 logprobs 接口。

---

## 8. 重要注意事项

### Post-training 后 PPL 会上升

经过 SFT（监督微调）和 RLHF（人类反馈强化学习）后，模型的 PPL 通常会**升高**。这是因为后训练教模型"如何完成任务"，而非"预测下一个 token"——任务完成能力增强了，但 token 预测能力可能下降。有人称之为"后训练导致的熵坍缩（entropy collapse）"。

### 量化（Quantization）会影响 PPL

降低模型数值精度的量化技术可能以不可预期的方式改变 PPL。

### PPL 不是万能指标

PPL 衡量的是语言建模能力，是下游表现的**代理指标**而非直接衡量。语言建模性能与下游表现虽然强相关，但并不能完全解释下游表现——这仍是活跃的研究领域。
