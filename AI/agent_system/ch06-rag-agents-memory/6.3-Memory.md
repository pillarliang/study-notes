# Memory（记忆系统）

> **核心定位**：RAG 和 Agent 系统都需要处理大量信息，往往超出模型的上下文长度限制。**记忆系统（Memory System）是对模型上下文的补充机制——它让模型能够保留和利用信息**，从而显著增强模型的能力。RAG 系统依赖记忆来存储增强上下文（augmented context），而 Agent 系统需要记忆来存储指令、示例、上下文、工具清单、计划、工具输出、反思结果等。虽然 RAG 和 Agent 对记忆的需求最大，但任何需要保留信息的 AI 应用都能从记忆系统中受益。

---

## 一、三种记忆机制

AI 模型通常有三种主要的记忆机制，它们与人类的记忆方式高度类比：


| 记忆类型                         | 说明                                                                     | 人类类比                                 | 特点               |
| ---------------------------- | ---------------------------------------------------------------------- | ------------------------------------ | ---------------- |
| **内部知识（Internal Knowledge）** | 模型自身就是一种记忆机制——它保留了训练数据中的知识。除非更新模型本身，否则内部知识不会改变。模型在所有查询中都能访问这些知识        | 你知道怎么呼吸——这是"刻在身体里"的知识，除非出了大问题，否则不会忘记 | 全局可用、不可变（除非重新训练） |
| **短期记忆（Short-term Memory）**  | 模型的**上下文（context）就是短期记忆。对话中的前序消息可以加入上下文，使模型能利用它们生成后续回答。短期记忆不跨任务持久化**   | 你正在做的事情中立即相关的信息，比如刚认识的人的名字           | 访问快、容量有限、任务结束即丢失 |
| **长期记忆（Long-term Memory）**   | 模型通过检索访问的**外部数据源**（如 RAG 系统中的数据库）即为长期记忆。它**可跨任务持久化**，且信息可以被删除而无需更新模型本身 | 书籍、笔记、电脑等外部存储工具——你不需要记住所有东西，需要时查阅即可  | 容量大、可持久化、可增删     |


### 如何选择记忆类型？

选择哪种记忆机制取决于数据的**使用频率**：

- **所有任务都需要的信息** → 通过训练或微调写入**内部知识**
- **偶尔需要的信息** → 存放在**长期记忆**中
- **当前任务立即需要的信息** → 放入**短期记忆**

> 这三种记忆机制构成了一个**信息层级结构（hierarchy of information）**，如同人类的记忆分层。

## 二、记忆系统的价值

为 AI 模型增加记忆系统有以下核心收益：

### 1. 管理会话内信息溢出（Manage Information Overflow Within a Session）

Agent 在执行任务过程中会获取大量新信息，可能超出模型的最大上下文长度。**溢出的信息可以存入长期记忆**，避免丢失。

### 2. 跨会话持久化信息（Persist Information Between Sessions）

- 一个 AI 教练如果每次对话都要用户重新讲述整个背景，那几乎毫无用处
- 一个 AI 助手如果总是忘记用户的偏好，使用体验会很糟糕
- 有了对话历史的记忆，Agent 可以实现**个性化**。例如，当你请求书籍推荐时，如果模型记得你之前喜欢《三体》，它就能推荐类似风格的书

### 3. 提升模型一致性（Boost Consistency）

如果你问一个主观问题两次（比如"给这个笑话打 1-5 分"），当模型能够引用自己之前的回答时，它更可能给出一致的答案。记忆系统让模型能够**校准后续回答**，保持前后一致。

### 4. 维护数据结构完整性（Maintain Data Structural Integrity）

文本天生是非结构化的，存储在文本模型上下文中的数据也是非结构化的。虽然你可以逐行将表格输入上下文，但无法保证模型理解这是一张表格。记忆系统能够存储结构化数据，维护数据的结构完整性。例如：

- Agent 可以利用 **Excel 表格**存储销售线索
- Agent 可以利用**队列（queue）**存储待执行的操作序列

---

## 三、记忆系统的两大功能

一个 AI 记忆系统通常包含两个核心功能：


| 功能                          | 说明                                                |
| --------------------------- | ------------------------------------------------- |
| **记忆管理（Memory Management）** | 管理什么信息应存储在短期记忆和长期记忆中——包括**添加**和**删除**操作           |
| **记忆检索（Memory Retrieval）**  | 从长期记忆中检索与当前任务相关的信息——本质上与 RAG 检索相同，因为长期记忆就是一种外部数据源 |

## 四、记忆管理策略

记忆管理并非 AI 应用独有——**内存管理（memory management）一直是所有数据系统的基石**，业界已经发展出大量高效利用内存的策略。

### 短期记忆与长期记忆的容量分配

长期记忆可用于存储短期记忆的**溢出（overflow）**。对于给定的查询，输入模型的上下文同时包含短期记忆和从长期记忆中检索的信息。因此，短期记忆的容量取决于为长期记忆检索结果预留了多少空间。

**示例**：如果上下文的 30% 预留给长期记忆检索结果，则模型最多能用 70% 的上下文容量作为短期记忆。当短期记忆达到阈值时，溢出部分被迁移到长期记忆中。

### 策略一：FIFO（先进先出）

最简单的策略——**最先加入短期记忆的信息最先被移出**。

- **实现方式**：随着对话变长，API 提供商（如 OpenAI）可能开始移除对话开头的内容。LangChain 等框架支持保留最近 N 条消息或最近 N 个 token
- **隐含假设**：在长对话中，早期消息与当前讨论相关性较低
- **致命缺陷**：这个假设可能严重错误。某些对话中，**最早的消息可能承载最重要的信息**——尤其是开头陈述对话目的的消息。FIFO 虽然实现简单，但可能导致模型丢失关键信息

> 基于使用频率的策略（如移除最不常用的信息）实现更困难——你需要一种方法来判断模型何时"使用"了某条信息。

### 策略二：去除冗余（Redundancy Removal）

人类语言天然包含冗余——这是为了增强清晰度、弥补潜在的误解。如果能自动检测冗余，记忆占用将显著减少。

#### 方法 A：摘要法

用**对话摘要（summary）**替代完整对话。摘要可由同一个模型或另一个模型生成。**摘要 + 命名实体追踪**已经能走很远。

Bae et al.（2022）更进一步——生成摘要后，将摘要与原始记忆中摘要遗漏的关键信息**合并**，构造新的精简记忆。具体做法是：开发一个分类器，对原始记忆中的每个句子和摘要中的每个句子，判断应保留哪一个、两个都保留、还是都丢弃。

#### 方法 B：反思法（Reflection Approach）

Liu et al.（2023）采用反思机制。每次行动之后，Agent 被要求完成两件事：

1. **反思**刚刚生成的信息
2. **决定**这条新信息应该：
  - **插入**记忆（全新信息）
  - **合并**到已有记忆中（补充信息）
  - **替换**某条旧信息（尤其当旧信息已过时、与新信息矛盾时）

### 处理矛盾信息

当遇到矛盾的信息片段时，存在几种处理方式：


| 策略            | 说明                                    |
| ------------- | ------------------------------------- |
| **保留较新的**     | 最简单的策略——假设新信息更准确                      |
| **让 AI 模型判断** | 让模型根据上下文判断保留哪一个                       |
| **两个都保留**     | 矛盾信息可能让 Agent 困惑，但也可能帮助它从**不同视角**进行推理 |


最终采用哪种策略取决于具体的应用场景。

## 五、记忆系统在 RAG 与 Agent 中的角色


| 系统        | 记忆的作用                                                               |
| --------- | ------------------------------------------------------------------- |
| **RAG**   | 记忆系统提供**增强上下文**——在多轮检索中，上下文会不断增长，记忆系统负责管理这些持续累积的信息                  |
| **Agent** | 记忆系统存储**指令、示例、上下文、工具清单、计划、工具输出、反思**等全部信息，是 Agent 追踪自身进度、实现多步推理的基础设施 |


> RAG 和 Agent 都是**基于 Prompt 的方法**——它们仅通过改变输入来影响模型质量，而不修改模型本身。记忆系统是两者管理和利用信息的共同基础组件。
