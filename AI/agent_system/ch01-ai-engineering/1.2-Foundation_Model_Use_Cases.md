# Foundation Model Use Cases（基础模型应用场景）

> **核心定位**：基础模型能构建的应用数量近乎无限——截至 2024 年 9 月，theresanaiforthat.com 列出了 16,814 个 AI 工具，覆盖 14,688 个任务和 4,803 个岗位。本节将这些应用归纳为八大类别，并分析企业与消费者场景的差异，帮助读者建立"AI 能做什么"的全景认知。

---

## 一、应用场景分类概览

不同调研机构对 AI 应用场景的分类各有不同：

| 来源 | 分类方式 |
|---|---|
| **AWS** | 客户体验、员工生产力、流程优化 |
| **O'Reilly 2024 调查** | 编程、数据分析、客户支持、营销文案、其他文案、研究、网页设计、艺术（8 类） |
| **Deloitte** | 按价值获取分类：成本削减、流程效率、增长、加速创新 |
| **Gartner** | 增加了 **business continuity**（业务存续）类别——不拥抱 AI 可能导致企业被淘汰。2023 年 Gartner 调查 2,500 名高管，7% 将业务存续作为拥抱生成式 AI 的动机 |

### AI 受影响最大的职业（Eloundou et al., 2023）

该研究将"暴露"定义为：AI 和 AI 驱动的软件能将完成某项任务所需的时间**减少至少 50%**。暴露度 80% 意味着该职业 80% 的任务都受到影响。

| 群组 | 职业 | 暴露度 |
|---|---|---|
| Human α（直接暴露于 AI 模型） | 口译员和翻译 | 76.5% |
| | 调查研究员 | 75.0% |
| | 诗人、词作者、创意写作者 | 68.8% |
| | 动物科学家 | 66.7% |
| | 公关专家 | 66.7% |
| Human β（暴露于 AI 驱动软件） | 调查研究员 | 84.4% |
| | 作家和作者 | 82.5% |
| | 口译员和翻译 | 82.4% |
| | 公关专家 | 80.6% |
| Human ζ（100% 暴露） | 数学家、报税员、金融量化分析师、作家、网页/数字界面设计师 | 100.0% |

几乎不受 AI 影响的职业：厨师、石匠、运动员。

### 作者调研的八大类别

作者通过采访 50 家企业 AI 策略 + 100 多个企业案例 + 分析 205 个 GitHub 500 星以上的开源 AI 应用，将应用归为八组：

| 类别 | 消费者场景示例 | 企业场景示例 |
|---|---|---|
| **编程（Coding）** | 编程辅助 | 编程辅助 |
| **图像与视频制作** | 照片/视频编辑、设计 | 演示文稿、广告生成 |
| **写作（Writing）** | 邮件、社交媒体、博客 | 文案、SEO 优化、报告、备忘录、设计文档 |
| **教育（Education）** | 辅导、论文评分 | 员工入职、技能提升培训 |
| **对话机器人** | 通用聊天、AI 伴侣 | 客户支持、产品副驾驶 |
| **信息聚合** | 摘要、Talk-to-your-docs | 摘要、市场调研 |
| **数据组织** | 图像搜索、Memex | 知识管理、文档处理 |
| **工作流自动化** | 旅行规划、活动策划 | 数据提取/录入/标注、线索生成 |

### 开源应用分布（205 个 GitHub 仓库）

| 类别 | 占比 |
|---|---|
| Coding | **30.4%** |
| Conversational bots | **26.5%** |
| Image and video production | 12.7% |
| Info aggregation | 12.7% |
| Workflow automation | 11.3% |
| Writing | 3.4% |
| Data organization | 1.5% |
| Education | 1.5% |

教育、数据组织、写作类应用在开源中占比低，不代表不热门——而是因为这些应用更多以闭源商业产品形式存在。

### 企业偏好：内部优先

a16z 2024 Growth 报告显示，企业更倾向于优先部署**内部应用**（Internal-facing），而非外部应用（External-facing）：

| 应用场景 | 部署到生产的企业占比 |
|---|---|
| 文本摘要（内部） | 62% |
| 企业知识管理（内部） | 60% |
| 客户服务（内部/外部） | 59% |
| 营销文案 | 53% |
| 软件开发 | 53% |
| 合同审查 | 45% |
| 外部聊天机器人 | 39% |
| 推荐算法 | 39% |

原因：内部应用允许企业在控制数据隐私、合规和灾难性失败风险的前提下积累 AI 工程经验。同时，虽然基础模型本身是开放式的，但许多基于其构建的应用实际上是**封闭式的分类任务**——更容易评估，风险更可控。

## 二、各场景详解

### 2.1 编程（Coding）——最热门的 AI 应用场景

在多项生成式 AI 调查中，编程都是**最受欢迎的应用场景**，原因有二：AI 擅长编程 + 早期 AI 工程师多为程序员。

**标志性成功**：GitHub Copilot——上线仅两年，年度经常性收入（ARR）就突破 **1 亿美元**。截至写作时，AI 编程创业公司已融资数亿美元（Magic 融资 3.2 亿、Anysphere 融资 6,000 万，均为 2024 年 8 月）。

**AI 编程的生产力提升**（McKinsey 数据）：

| 任务类别 | 时间减少幅度 |
|---|---|
| 代码文档撰写 | **45-50%** |
| 代码生成 | **35-45%** |
| 代码重构 | **20-30%** |
| 高复杂度任务 | **<10%** |

关键洞察：AI 对**简单任务**帮助最大，对高复杂度任务帮助有限。开发者反馈：AI 在前端开发的表现明显优于后端开发。

**细分编程工具示例**：
- 网页/PDF 结构化数据提取（AgentGPT）
- 自然语言转代码（DB-GPT、SQL Chat、PandasAI）
- 截图转网站代码（draw-a-ui）
- 跨语言/框架翻译（GPT-Migrate、AI Code Translator）
- 文档自动生成（Autodoc）
- 测试生成（PentestGPT）
- Commit message 生成（AI Commits）

**AI 会取代软件工程师吗？**

两极观点：
- **取代派**：NVIDIA CEO Jensen Huang 预测 AI 将取代人类软件工程师；AWS CEO Matt Garman 预测大多数开发者将停止编码（工作性质将改变而非消失）
- **不会派**：许多工程师坚信自己不可被取代（技术和情感双重原因）

**现实判断**：无论是否取代，AI 确定可以让工程师大幅提高生产力——企业可以用更少的工程师完成更多工作。AI 还可能冲击外包行业，因为外包的通常是核心业务之外的较简单任务，恰好是 AI 最擅长的。

### 2.2 图像与视频制作

AI 的概率性本质使其天然适合创意工作。代表性创意 AI 产品：

| 产品 | 领域 |
|---|---|
| Midjourney | 图像生成 |
| Adobe Firefly | 照片编辑 |
| Runway、Pika Labs、Sora | 视频生成 |

- **Midjourney**：2023 年底，成立仅一年半就达到 **2 亿美元** ARR
- 2023 年 12 月，Apple App Store 图形与设计类 Top 10 免费应用中，**一半名称含有 "AI"**
- AI 生成个人头像用于社交媒体（LinkedIn、TikTok）已成常态。2019 年 Facebook 还在因安全原因封禁 AI 头像；到 2023 年，社交媒体应用已主动提供 AI 头像生成工具

**企业应用**：广告和营销领域拥抱 AI 最快——直接生成宣传图片/视频、头脑风暴创意、生成多版本广告进行 A/B 测试、根据季节/地区自动调整素材（秋天换叶色、冬天加雪景）。

### 2.3 写作（Writing）

LLM 本质就是文本补全训练，因此擅长写作并不意外。

**MIT 研究**（Noy and Zhang, 2023）：让 453 名受过大学教育的专业人士完成职业写作任务，随机将一半暴露于 ChatGPT：
- 使用 ChatGPT 的人完成时间减少 **40%**
- 输出质量提升 **18%**
- ChatGPT 对写作意愿较低的人帮助更大（缩小了质量差距）
- 实验后 2 周、1.6 个月，受试者在真实工作中使用 ChatGPT 的概率分别是未暴露组的 **2 倍**

**消费者场景**：
- 情绪润色邮件、bullet points 扩展为完整段落
- 学生用 AI 写论文（ChatGPT 宕机时 OpenAI Discord 挤满抱怨无法完成作业的学生）
- 创业公司用 AI 生成儿童书、言情小说、奇幻小说，且可交互——读者选择影响剧情
- AI 生成的低质量旅游指南大量涌入 Amazon（New York Times 2023 年报道）

**AI 与 SEO**：AI 特别擅长 SEO，因为训练数据本身大量来自 SEO 优化过的互联网内容。这催生了新一代"内容农场"——用 AI 生成垃圾网站骗取 Google 流量和广告收入。2023 年 6 月，NewsGuard 在 141 个热门品牌中发现近 400 条广告出现在 AI 生成的垃圾网站上，其中一个垃圾网站日产 **1,200 篇**文章。

**企业场景**：销售、营销、日常团队沟通中广泛使用。AI 帮助撰写绩效报告、冷启动邮件、广告文案、产品描述。HubSpot、Salesforce 等 CRM 工具已内置 AI 生成功能。

**写作辅助工具**：Google Docs、Notion、Gmail 都已集成 AI。Grammarly 通过微调模型让用户写作更流畅、连贯、清晰。

### 2.4 教育（Education）

**现状与争议**：纽约市公立学校和洛杉矶联合学区最初封禁 ChatGPT 防止作弊，但仅几个月后就**撤回了禁令**。

**AI 在教育中的真正机会**——个性化教学：
- 为每个学生生成个性化讲义和学习计划
- 听觉学习者：AI 朗读材料
- 视觉学习者：AI 调整可视化内容（如增加动物图片）
- 偏好代码的学生：AI 将数学公式翻译为代码
- 语言学习：AI 可以扮演不同角色进行练习对话

**Duolingo 四阶段模型**（Pajak and Bicknell, 2022）：

| 阶段 | 人类工作量 | AI 工作量 | 说明 |
|---|---|---|---|
| Stage 1: 课程设计 | 高 | 低 | AI 辅助有限 |
| Stage 2: 原始内容创建 | 中 | 中 | AI 可参与内容生成 |
| Stage 3: 练习题创建 | 中 | 中高 | AI 大量参与 |
| Stage 4: **课程个性化** | 低 | **高** | AI 获益最大的阶段 |

**AI 在教育中的其他应用**：
- 自动出题（选择题 + 开放题）并评分
- 充当辩论伙伴（展示不同观点的能力优于普通人）
- Khan Academy 提供 AI 教学助手和课程助手
- 创新方式：教师让学生寻找 AI 生成论文中的错误

**行业冲击案例**：Chegg（帮学生做作业的公司）股价从 ChatGPT 上线前的 $28 暴跌至 2024 年 9 月的 **$2**，因为学生转向了 AI。

**风险与机会并存**：AI 可能取代许多技能，但也可以成为**任何技能的导师**——帮助人们快速入门，然后自主深入学习以超越 AI。

### 2.5 对话机器人（Conversational Bots）

对话机器人用途极其广泛：查找信息、解释概念、头脑风暴、模拟人格（AI 伴侣、AI 男/女朋友）。数字伴侣在极短时间内爆红，许多人与 AI 聊天的时间已超过与真人聊天。

**研究应用**：一组对话机器人可以模拟一个社会，用于研究社会动态（Park et al., 2023）。

**企业应用**：
- 最主流的是**客户支持机器人**——节省成本、加速响应、7×24 小时服务
- **产品副驾驶**（Product Copilot）——引导用户完成复杂流程（报保险、做税务、查公司政策）
- 语音助手（Google Assistant、Siri、Alexa）也是对话式 AI，但整合生成式 AI 进展较慢（可能因为语音界面的质量标准和合规要求更高）

**游戏中的对话 AI**：AI 驱动的智能 NPC（非玩家角色）是新兴应用。传统 NPC 只有预设台词和有限动作，AI 可让 NPC 变得更智能，从根本上改变 The Sims、Skyrim 等游戏的动态体验。代表项目：NVIDIA 的 Inworld 和 Convai 演示。

### 2.6 信息聚合（Information Aggregation）

成功常取决于**筛选和消化有用信息**的能力，但邮件、Slack 消息、新闻的信息洪流让人不堪重负。

根据 Salesforce 2023 年调查，**74%** 的生成式 AI 用户使用它来**提炼复杂观点和总结信息**。

**消费者场景**——Talk-to-your-docs：
- 处理合同、披露文件、论文，以对话方式检索信息
- AI 总结网站内容、做研究、生成专题报告

**企业场景**：
- **Instacart** 内部 prompt 市场中最受欢迎的模板之一是 "Fast Breakdown"——总结会议纪要、邮件、Slack 对话，提取事实、未决问题、行动项，并自动分配到项目跟踪工具
- AI 可识别潜在客户信息、竞品分析
- 信息聚合与数据组织密切相关

### 2.7 数据组织（Data Organization）

未来确定无疑的一点：**数据只会越来越多**——照片、视频、日志、PDF、合同，每年数以十亿计。这些非结构化或半结构化数据必须被组织成可搜索的形式。

**AI 的作用**：
- 自动为图片和视频生成文本描述
- 文本查询匹配视觉内容（Google Photos 已在使用）
- Google Image Search 更进一步：若无匹配图片，直接**生成**一张
- 数据可视化、异常检测、收入预测

**企业场景——智能文档处理（IDP）**：
- 从信用卡、驾照、收据、票据、邮件签名中自动提取结构化信息
- 从合同、报告、图表中提取复杂数据
- IDP 行业预计到 2030 年将达到 **128.1 亿美元**，年增长率 32.9%

### 2.8 工作流自动化（Workflow Automation）

AI 的终极目标是尽可能多地自动化。

**消费者场景**：订餐厅、退款申请、旅行规划、填表

**企业场景**：
- 线索管理、发票处理、报销、客户请求管理、数据录入
- 一个特别有趣的应用：**用 AI 合成训练数据来改进模型本身**——用 AI 为数据生成标签，再由人类审核标签质量（详见 Chapter 8）

**通向 Agent 的路径**：完成工作流自动化需要**调用外部工具**——搜索引擎查电话、拨号、日历添加约会。能够规划和使用工具的 AI 被称为 **Agent**（智能体），详见 Chapter 6。

## 三、关键洞察总结

| 洞察 | 要点 |
|---|---|
| **应用可跨类别** | 基础模型是通用的，单个应用可能同时属于多个类别（如一个 bot 既能陪聊也能聚合信息） |
| **编程和对话机器人**占主导 | 在开源应用中合计占比 **57%** |
| **企业偏好内部应用** | 风险更可控、更易评估、有助于积累 AI 经验 |
| **AI 对简单任务帮助最大** | 编程中：文档撰写提效 45-50%，但高复杂度任务 <10% |
| **AI 的最大价值不是取代而是增强** | 让每个人都能更高效，同时也可以成为任何技能的导师 |
| **应用惊喜不断** | 如同互联网早期无人预见社交媒体会成为杀手级应用，AI 的杀手级应用可能尚未出现 |
